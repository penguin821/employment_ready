pch - 정적 라이브러리로 생성 (최종 패키지에 라이브러리가 같이 포함되어서 따로 관리할 필요 없음), 
	동적연결 라이브러리 아님( .dll 파일로 패키지와 따로 늘 관리해줘야함) 
	
	해당 프로젝트에 자주 쓰이는 헤더들을 모아서 미리 컴파일할 헤더들 모아둠
	하지만 해당 애가 PCH 답게 작동하게 만들기 위에선 해당 프로젝트 속성 들어가서
	C/C++ 미리 컴파일된 헤더에 미리 컴파일 사용 안함을 사용함으로 바꾸고
	미리 컴파일된 헤더 파일을 stdafx.h 말고 pch.h로 바꿔주면 된다
	그리고 pch.cpp 파일 속성 들어가서 똑같은 경로에 미리 컴파일된 헤더를 사용에서 만들기로 바꿔주면 끝

	그럼 다른 프로젝트에 있는 PCH 같은 헤더를 어떻게 적용시키냐
	사용하고자 하는 프로젝트의 속성에 들어가서 c/c++ - 일반 = 추가 포함 디렉터리
	링커 - 일반 - 추가 라이브러리 디렉터리 에다가 각각 넣어줘서 설정하는 방법도 있지만

	VC++ 디렉터리에서 포함 디렉터리(라이브러리가 참조하는 파일들 위치)
	$(SolutionDir(같은 솔루션안에 있다는 가정하에))프로젝트명\

	라이브러리 디렉터리(라이브러리 위치) 설정 해주는게 조금 더 빠르다.
	$(SolutionDir(같은 솔루션안에 있다는 가정하에))라이브러리모아놓은폴더명\
	
	이후 링커-입력-추가 종속성에다가 .lib들을 포함하는 방법도 있지만 그렇게 하면
	라이브러리 하나하나 다 매번 넣어줘야하므로
	그냥 pch 에다가 만들어주는 방법도 있음

우클릭 시작프로젝트 설정 (단축키 A)


binary 폴더 - 실행 결과물들 모아놓은 폴더
해당 폴더로 결과물 경로 바꾸려면 프로젝트 우클릭 속성 일반 출력 디렉터리 경로 바꿔줄것
$(SolutionDir)<요기에 폴더명\>$(Configuration)\

Libraries 폴더 - 라이브러리들만 모아놓은 폴더
