전역변수로 const int 로 선언해서 쓰는거랑
enum으로 열거하는거랑 차이점이 뭐냐

이넘은 함수라서 스택 메모리에 임시로 올라갔다 사라지지만
전역변수 const int는 실행하자고 끝날때까지 메모리를 잡아버림

거기다가 const int로 선언하면 상수명이랑 상수값이 둘 다 메모리로 잡히지만
enum은 상수값만 저장되고 상수명은 껍데기처럼 존재함

#define 상수명 상수값
으로도 가능함
빌드는 1전처리 2컴파일 3링크 과정으로 이루어짐
전처리에서 기초적인 코드들을 실행하고
컴파일에서 본 코드들을 실행하고 결과물을 만들고
링크에서 결과물을 뽑아주고 파일을 만드는 작업

#include <> 하면
전처리 단계에서 괄호 안에 있는 파일을 찾아서 그대로 복붙해주세요 라는 뜻

상수화의 용도보단 코드 바꿔치기로 더 많이 쓴다

코드 길어지면 디버깅 포인트도 안잡히고 하니
디파인은 상수화 방법의 후순위로 쓸것
